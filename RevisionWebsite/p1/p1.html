  • Types of Testing

    Blackbox: carried out independently of the code within the program, look at program specification and devise appropriate test data.
    White box: testing is dependent on the code logic, mostly from the program structure rather than functionality. every path that the code can take is checked at least once.
    Alpha Testing: carried out by the in house testing team, this is where checks are carried out to see if there is missing functionality from the specification of the program.
    Beta Testing: The game is released in its current state to the public, this is where the users will report any problems to the developers.

  • Development Methodologies

    Waterfall Cycle: every step of the development process is carried out from beginning to end, without any interruptions.
    Spiral Model: Uses the same structure as the Waterfall cycle, but performs these steps iteratively, each loop generates a working prototype of the software
    Agile Modelling: Goes against the first two methodologies, as a more iterative and non-linear approach is taken, quick protoypes are designed and created, then to be evaluated by a customer, then improve the protoype and create the final system.
    Extreme Programming: Another agile methodology, this one in particular focuses on the customer feedback as a priority
    RAD: Promotes the idea of rapid prototyping, aims to complete a good enough solution within a small timeframe.

  • Merits and Drawbacks for each methodology

    Waterfall: Suitable for small/simple projects, but low user involvement is a great drawback.
    Spiral/Agile: avoid formal requirements like waterfall cycle, protoypes are much easier to evaluate compared to a formal specification document
    Extreme/RAD: Great methodology for large projects, where there is a danger of getting sidetracked by suggested improvements
