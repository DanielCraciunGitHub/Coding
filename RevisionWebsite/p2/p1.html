  • Arrays, Tuples, Records

    Arrays: A composite data structure, which is mutable, and stores data of the same data type, where each item can be accessed by index.
    Tuples: A composite data structure, which is immutable, but stores data of any types, and also accessed by index.
    Records: A composite data structure, which is mutable, can store elements of any data type, but accessed by name.

  • Queues

    A FIFO data structure, similar to a queue in real life

  • Lists and Linked lists

    List: A dynamic data structure that behaves similarly to an array, but it varies in size, unlike an array which has a set size
    Linked List: A dynamic data structure that consists of nodes and pointers, where each node will point to the next node within the linked list, and the last pointer points to null

  • Stacks

    A LIFO data structure, where the last element added into the stack is the first element out of the stack

  • Hash Table

    A data structure that can be implemented as an array of linked lists, and a hash value is calculated on a value to add to the hask table, where the hash value is the location of the item within the hash table

  • Graphs

    A data structure that is very similar to a linked list, but rather than nodes following a linear pointing structure, and node can point to any other node within the graph.

  • Tree

    A data structure similar to a graph, but consists of a root node, from which every other node stems from it
